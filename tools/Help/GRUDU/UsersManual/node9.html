Manual installation

If you want to compile GRUDU sources, you need to install the Java build tool Ant 2.1. Then simply execute the following command: ant GRUDU for the compilation. If the compilation succeed, you will get a new jar file GRUDU.jar representing the program. Launching GRUDU can be done like a typical jar file: java -jar GRUDU.jar.

It is preferable that your first step with GRUDU is to configure it. The configuration window contains two tabs, the first corresponding to the user parameters while the second tab is relative to GRID'5000 sites and cluster.

The figure 4.1 represents the first tab of configuration windows, where we can found the following parameters:

  • Preferred access point: this is GRID'5000 site GRUDU will use to access to all the platform. Your account in this site must contain your private ssh key file.
  • Username: your GRID'5000 username.
  • Private SSH key file: your private ssh key file.
  • Password: the passphrase of your ssh key.

Figure 2.6: Configuration window: First tab

The second configuration tab shown in figure 4.2 allows to configures the following parameters:

  • Enabling/Disabling a cluster: by default all clusters are disabled.
  • Kadeploy partition for each cluster.
