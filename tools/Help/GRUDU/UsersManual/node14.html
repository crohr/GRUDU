Configuration files

GRUDU uses several configuration files saved in .diet directory which is located at the root of you home directory.

[[:]:!:}[[:]:!:}Cluster configuration file: g5k_cluster.xml 

<g5k>
<cluster
id="Lyon--Capricorne"
enable="false"
xda=""
batch_schedulers=""
/>
...
</g5k>

As shown before, this configuration file defines for each cluster the following parameters:

  • id: cluster identifier, typically it is in the form "site-cluster".
  • enable: determine if the user wants to use this cluster or not.
  • xda: Kadeploy partition.
  • batch_scheduler: oar1 or oar2.

[[:]:!:}[[:]:!:}GRID'5000 configuration file: g5k.xml 

<g5k>
  <preferredAccesPoint host="acces.lyon.grid5000.fr" />
  <username id="pdupont" />
  <sshkey file="/path/to/my/ssh/private/key" password="my_passphrase" />
  <goDietScratch dir="" />
  <goDiet debug="" useUniqueDirs="" saveStdOut="" saveStdErr=""  />

<!-- G5K Sites -->
...
</g5k>

This file defines the global information used to log in GRID'5000. Two nodes are not used by GRUDU but present in these file (goDietScratch and goDiet), and they won't be described.

Here are the parameters that should be supplied:

  • preferredAccesPoint: the node has an attribute named host. This attribute have to be the name of one of the access frontal of GRID'5000.
  • username: the node has an attribute names id that is the login of the user.
  • sshkey: the node has two attribute, file which is the file storing the ssh private key file, and password which is th passphrase used with this key.
