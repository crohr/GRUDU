Resources allocation

[[:]:!:}[[:]:!:}Using OAR interface

The most used operation is probably resources allocation. In GRID'5000, this operation can be done by OAR system. GRUDU provides an easy way to manipulate OAR. The resources allocation windows on Figure 4.1 shows a map of France with GRID'5000 sites 4.1 and jobs characteristics (time, queue, oargridsub behaviour, the script to launch). These information are presented on the first tab of the window. The second tab provides the definition of the properties for the different clusters Since some sites include more than one cluster, you have to click first on the site, and then to select the number of desired hosts. When selecting resources numbers, the map displays the total number of requested resources for each site4.2. Jobs characteristics are:

  • Time parameters: date and reservation walltime. The starting time can be specified manually, or through the use of a calendar for the day and through boxes where you can specify the hour, the minute and the second. For the walltime you have to define the number of hours and the minutes of your reservation.
  • Queue: default or Kadeploy.
  • OARGridSb behaviour: the user can specify if the reservation should be done with the OARGridSub behaviour, i.e. when the user chooses to realize several reservations on different sites, if one fails, all the previous successful reservations are deleted.
  • A script to launch: The user can specify a script that will be launched in order to be executed on the reserved nodes. The reservation will be stopped when the script ends.
